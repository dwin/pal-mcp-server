## Conversation Continuation

Clink supports two methods for continuing conversations:

1. **continuation_id** (PAL memory): Use for cross-tool continuation within PAL. Pass the `continuation_id` from any PAL tool response to maintain context across different tools (chat, analyze, codereview, clink, etc.).

2. **cli_session_id** (CLI native): Use for CLI-specific session resumption. Pass the `cli_session_id` from a clink response to resume the same CLI session using its native --resume capability. This preserves the CLI's internal conversation state.

Both can be used together for hybrid continuation - PAL remembers the cross-tool context while the CLI maintains its own session state.
